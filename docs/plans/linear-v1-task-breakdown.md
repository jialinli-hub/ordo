## 6. 依赖关系与关键路径

- 关键路径：Schema -> Issue 核心 -> 视图能力 -> 自动化引擎 -> 集成能力 -> 稳定化上线。
- 强依赖：
  - S1-W3 必须先于 S1-W4/S1-W5。
  - S2-W7 自动化引擎先于 S2-W8 集成联动。
  - S2-W8 先于 S2-W9 的外部 Webhook 发布能力。

## 7. 每周交付节奏（建议）

- 周一：确认本周目标与风险。
- 周三：中期检查（阻塞清理）。
- 周五：功能验收 + 风险复盘。
- 每周输出：已完成清单、下周计划、风险追踪表。

## 8. 风险清单（执行版）

| 风险 | 影响 | 预防措施 | 应急策略 |
|---|---|---|---|
| 编号冲突 | Issue 创建失败 | 计数器锁与事务 | 人工修复脚本 + 补偿任务 |
| 规则误触发 | 状态异常 | 条件白名单与幂等 | 回滚任务 + 操作审计 |
| 集成抖动 | 同步失败 | 重试与死信队列 | 定时补偿 + 告警升级 |
| 查询变慢 | 体验下降 | 索引与慢 SQL 治理 | 缓存与分页策略收紧 |

## 9. Definition of Done（DoD）

- 功能、测试、文档三项完成且通过评审。
- 覆盖核心异常场景并具备回归用例。
- 监控指标可观测，关键日志可追踪。
- 无阻塞级缺陷，发布条件达标。
